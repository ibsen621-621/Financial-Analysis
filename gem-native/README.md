# 三表穿透分析 Gem · 原生版（无后端）

> **本目录是"Gem 原生版"**：所有计算与判定由 Gemini Gem 模型在对话中直接完成，无需部署任何后端服务。
>
> 与仓库根目录 README 描述的「API 版」的区别：API 版需要自行托管 FastAPI 服务（`app.py`），Gem 通过 Function Calling 回调后端计算；原生版把全部口径、公式、规则阈值写入 Gem 的 Knowledge 文件，对话即计算，零运维。

---

## 部署三步骤

### 第一步：粘贴指令

打开 [Google Gemini Gems](https://gemini.google.com/gems)，新建一个 Gem，把 `gem-native/gem_instructions.md` 的**全部内容**复制粘贴到 **Instructions（指令）** 输入框，保存。

### 第二步：上传知识文件

在同一个 Gem 的 **Knowledge（知识库）** 栏，上传以下 4 个文件（全在 `gem-native/knowledge/` 目录下）：

| 文件 | 内容说明 |
|---|---|
| `metrics.md` | 全部指标计算口径（公式 + `safe_div` 语义） |
| `rulebook.md` | 7 条风险规则（R1–R7）及触发阈值 |
| `cross_check_and_verdict.md` | 三表勾稽规则 + 最终裁决逻辑 |
| `input_fields.md` | 输入字段字典（中英文名 + 必填说明） |

### 第三步：输入数据，得到报告

用 `gem-native/input_template.md` 里的模板填入目标公司的财务数据，把填好的内容粘贴给 Gem，即可获得一页式三表穿透分析报告。

---

## 何时用哪个版本

| 维度 | 原生版（本目录） | API 版（仓库根目录） |
|---|---|---|
| **后端依赖** | 无，零运维 | 需部署 FastAPI + 公网可达服务 |
| **计算精度** | 由 Gem 模型按固定协议手工计算 | Python 精确浮点计算 |
| **口径透明度** | Gem 可在对话中解释每条判定依据 | 口径锁在 Python，Gem 只拿到结果 |
| **适用场景** | 个人/团队快速分析、无服务器环境 | 需要程序化批量处理、精确复现的场景 |
| **部署成本** | 贴指令 + 传 4 个 md 文件，5 分钟完成 | 需要服务器、证书、鉴权、运维 |

## 验证样例

`examples/demo_company_5y.json`（仓库根目录）是一份示例公司财报（文件名含"5y"为命名惯例，实际数据覆盖 2021–2023 年三个年度），可用于验证 Gem 是否按正确口径输出报告。

**验证方法**：直接把 `demo_company_5y.json` 的完整 JSON 内容粘贴给已配置好的 Gem，检查输出的指标值和最终判断是否与预期一致。

预期结果（基于示例数据）：
- 2023 年触发 R5（商誉/净资产 = 0.30，商誉/核心利润偏高）
- 2023 年触发 R7（2 年以上应收坏账覆盖率 30/140 ≈ 0.21 < 0.30）
- 获现率连续下滑，可能触发 R2
- 最终判断：**回避**（高风险规则 ≥ 2）
