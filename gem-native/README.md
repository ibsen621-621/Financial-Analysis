# 三表穿透分析 Gem · 原生版（无后端）

> **本目录是"Gem 原生版"**：所有计算与判定由 Gemini Gem 模型在对话中直接完成，无需部署任何后端服务。
>
> 与仓库根目录 README 描述的「API 版」的区别：API 版需要自行托管 FastAPI 服务（`app.py`），Gem 通过 Function Calling 回调后端计算；原生版把全部口径、公式、规则阈值写入 Gem 的 Knowledge 文件，对话即计算，零运维。

---

## 部署三步骤

### 第一步：粘贴指令

打开 [Google Gemini Gems](https://gemini.google.com/gems)，新建一个 Gem，把 `gem-native/gem_instructions.md` 的**全部内容**复制粘贴到 **Instructions（指令）** 输入框，保存。

### 第二步：上传知识文件

在同一个 Gem 的 **Knowledge（知识库）** 栏，上传以下 **5 个文件**（全在 `gem-native/knowledge/` 目录下）：

| 文件 | 内容说明 |
|---|---|
| `metrics.md` | 全部指标计算口径（公式 + `safe_div` 语义） |
| `rulebook.md` | 7 条风险规则（R1–R7）及触发阈值 |
| `cross_check_and_verdict.md` | 三表勾稽规则 + 最终裁决逻辑 |
| `input_fields.md` | 输入字段字典（中英文名 + 必填说明） |
| `deep_analysis_framework.md` | 深度模式 7 大模块的分析要点、解读话术模板与趋势判断标准 |

### 第三步：输入数据，得到报告

用 `gem-native/input_template.md` 里的模板填入目标公司的财务数据，把填好的内容粘贴给 Gem，即可获得分析报告。

## 两种输出模式

本 Gem 支持两种输出模式，**共用全部计算与规则判定协议**，仅在输出格式上不同：

### 标准模式（默认）

- **触发方式**：直接粘贴数据，无需特殊指令。
- **产出内容**：A–F 六节一页式投研卡，包含摘要（≤ 300 字）、指标面板表、三表勾稽诊断、风险清单、最终判断和缺失数据清单。
- **适用场景**：快速获取财务风险概览，适合高频使用场景。

### 深度分析模式

- **触发方式**：在粘贴数据的同时（或之后），输入包含以下任一关键词的请求：
  - 中文：`深度分析`、`深度模式`、`详细报告`、`深度报告`
  - 英文：`deep`
  - 示例：「请对这份数据进行深度分析」或「用深度模式分析」
- **产出内容**：
  1. **执行摘要**：3–5 段自然语言概述底子/面子/日子/主要风险/最终判断
  2. **① 战略定位**：经营性与投资性资产占比及多年趋势
  3. **② 资产质量**：现金效率、应收账龄、存货结构、商誉占比
  4. **③ 资本结构**：供应链话语权、有息债务压力、预付款水平
  5. **④ 盈利质量**：核心利润趋势、毛利率、三费结构拆解
  6. **⑤ 现金质量**：获现率、投资回报验证、筹资与分红匹配度
  7. **⑥ 风险结论**：R1–R7 每条风险的「现象→机理→影响→建议复核项」四段式展开
  8. **⑦ 最终判断**：裁决结果 + 底子-面子-日子三表一致性论证
  9. **附录**：缺失数据清单
  - 每个模块均包含「年度指标小表 + 分析叙事 + 趋势解读」三要素。
- **适用场景**：需要深入叙事分析、趋势拆解、风险机理解读的场合，适合撰写完整投研报告或向他人汇报分析结果。

## 何时用哪个版本

| 维度 | 原生版（本目录） | API 版（仓库根目录） |
|---|---|---|
| **后端依赖** | 无，零运维 | 需部署 FastAPI + 公网可达服务 |
| **计算精度** | 由 Gem 模型按固定协议手工计算 | Python 精确浮点计算 |
| **口径透明度** | Gem 可在对话中解释每条判定依据 | 口径锁在 Python，Gem 只拿到结果 |
| **适用场景** | 个人/团队快速分析、无服务器环境 | 需要程序化批量处理、精确复现的场景 |
| **部署成本** | 贴指令 + 传 5 个 md 文件，5 分钟完成 | 需要服务器、证书、鉴权、运维 |

## 验证样例

`examples/demo_company_5y.json`（仓库根目录）是一份示例公司财报（文件名含"5y"为命名惯例，实际数据覆盖 2021–2023 年三个年度），可用于验证 Gem 是否按正确口径输出报告。

**验证方法**：直接把 `demo_company_5y.json` 的完整 JSON 内容粘贴给已配置好的 Gem，检查输出的指标值和最终判断是否与预期一致。

预期结果（基于示例数据）：
- 2023 年触发 R5（商誉/净资产 = 0.30，商誉/核心利润偏高）
- 2023 年触发 R7（2 年以上应收坏账覆盖率 30/140 ≈ 0.21 < 0.30）
- 获现率连续下滑，可能触发 R2
- 最终判断：**回避**（高风险规则 ≥ 2）
