# 深度分析框架（deep_analysis_framework.md）

> **本文件仅指导叙事与解读**，为深度模式（deep mode）的 7 大模块提供分析要点、解读话术模板和趋势判断标准。
>
> - 所有数值计算仍以 `metrics.md` 为准。
> - 所有风险阈值仍以 `rulebook.md` 为准。
> - 三表勾稽与最终裁决仍以 `cross_check_and_verdict.md` 为准。
> - **本文件不引入任何新公式、新指标或新阈值**，所引用的指标键名均来自 `metrics.md`，字段名均来自 `input_fields.md`。
>
> 本文件与仓库根目录 `README.md`「附：分析方法论（张新民框架）」（第 2–6 节）保持思想一致，使用底子（资产）-面子（利润）-日子（现金）三维框架。

## 模块一：战略定位

### 使用指标

| 指标键 | 来源 |
|---|---|
| `operating_assets` | 输入字段（若未直接提供，则按 `metrics.md` 合计字段派生协议正列举加总） |
| `investing_assets` | 输入字段（若未直接提供，则按 `metrics.md` 合计字段派生协议正列举加总） |
| `op_vs_inv_assets` | `metrics.md`：`operating_assets ÷ investing_assets` |
| `total_assets` | 输入字段，用于计算占比 |

### 分析要点

1. **经营性资产占比**（`operating_assets ÷ total_assets`）反映企业主业资源投入程度。长期维持在较高水平（如 >70%）说明企业以经营为主，主业驱动型。
2. **投资性资产占比**（`investing_assets ÷ total_assets`）过高且持续上升，说明企业战略向投资驱动倾斜，需关注投资回报与主业空心化风险。
3. **`op_vs_inv_assets` 比值**趋势：比值持续下降（投资性资产占比上升），可能预示战略转向；比值上升说明主业强化。

### 健康信号 vs 异常信号

| 信号类型 | 特征 |
|---|---|
| 健康 | 经营性资产占比稳定且较高；`op_vs_inv_assets` 比值稳定或上升 |
| 观察 | 投资性资产占比小幅上升，但经营性资产绝对值仍在增长 |
| 异常 | 投资性资产占比快速上升，经营性资产占比持续下降；或合计与所有对应分项字段**均缺失**（此时无法判断战略取向） |

### 框架解读

本模块对应张新民框架「底子—资产端：经营性资产 vs 投资性资产」维度，判断企业资产的「主业含量」——即资产中有多少是真正用于主营业务运营的，有多少是金融性或投资性持有的。

> **口径说明**：经营性资产（`operating_assets`）按张新民口径已剔除商誉（商誉由并购形成、不直接创造营业收入），因此 `op_vs_inv_assets` 比值衡量的是**核心主业资产**的主导程度，评估结果更能反映真实经营效率。`goodwill` 字段保留并单独通过 `goodwill_to_equity`、`goodwill_to_core_profit` 及 R5 规则分析并购风险。

## 模块二：资产质量

### 使用指标

| 指标键 | 来源 |
|---|---|
| `cash_ratio` | `metrics.md`：`cash ÷ total_assets` |
| `cash` | 输入字段 |
| `ar_to_rev` | `metrics.md`：`ar ÷ revenue` |
| `ar_aging_over_2y` | 输入字段（附注） |
| `ar_provision` | 输入字段（附注） |
| `ar_provision_coverage` | `metrics.md`：`ar_provision ÷ ar_aging_over_2y` |
| `inv_to_cogs` | `metrics.md`：`inventory ÷ cogs` |
| `inventory_provision` | 输入字段 |
| `goodwill_to_equity` | `metrics.md`：`goodwill ÷ total_equity` |
| `goodwill_to_core_profit` | `metrics.md`：`goodwill ÷ core_profit` |

### 分析要点

**现金效率**
- 货币资金规模是否与经营规模匹配：过低有流动性风险，过高且长期闲置可能是受限资金或关联方占用（联系 R1 四高异常）。
- `cash_ratio` 趋势：持续上升但伴随高有息负债，是资金受限或循环融资的典型信号。

**应收账款质量**
- `ar_to_rev` 趋势：持续上升说明收款效率下降，结合 `inv_to_cogs` 双升可能触发 R4（渠道压货嫌疑）。
- `ar_aging_over_2y`：账龄 2 年以上应收账款回收概率极低，是实质性坏账风险的直接指标。
- `ar_provision_coverage`（= `ar_provision ÷ ar_aging_over_2y`）< 0.30 触发 R7，说明计提严重不足，资产质量虚高。

**存货结构**
- `inv_to_cogs` 趋势：持续上升说明存货积压加剧，或成本下降而存货未同步消化。
- `inventory_provision`：跌价准备是否充足，若规模大但计提比例低，潜在减值风险高。

**商誉风险**
- `goodwill_to_equity` ≥ 0.30 且 `goodwill_to_core_profit` ≥ 3.0 同时触发 R5（商誉减值压力）。
- 商誉绝对值持续增大说明并购扩张仍在持续，需追踪并购标的业绩承诺完成情况。

### 健康信号 vs 异常信号

| 信号类型 | 特征 |
|---|---|
| 健康 | `ar_to_rev` 稳定或下降；`inv_to_cogs` 稳定；`ar_provision_coverage` ≥ 0.30；商誉占比低或为零 |
| 观察 | `ar_to_rev` 或 `inv_to_cogs` 单项上升，但获现率尚可；商誉占比接近阈值 |
| 异常 | 双升触发 R4；坏账覆盖率 < 0.30 触发 R7；商誉双指标触发 R5；货币资金高但有息负债也高触发 R1 |

### 框架解读

本模块对应张新民框架「底子—核心资产质量穿透」维度。资产负债表上列示的资产是否真实可变现，是判断"底子"成色的关键。

## 模块三：资本结构

### 使用指标

| 指标键 | 来源 |
|---|---|
| `ap_vs_inventory` | `metrics.md`：`ap ÷ inventory` |
| `ap` | 输入字段 |
| `prepayments` | 输入字段 |
| `debt_ratio` | `metrics.md`：`(short_debt + long_debt) ÷ total_assets` |
| `short_debt` | 输入字段 |
| `long_debt` | 输入字段 |
| `fin_exp_ratio` | `metrics.md`：`financial_exp ÷ revenue` |

### 分析要点

**供应链话语权**
- `ap_vs_inventory` > 1：应付账款大于存货，说明企业能充分利用供应商信用，对供应商的议价能力强。
- `ap_vs_inventory` 趋势下降：议价能力可能在减弱（如行业话语权转移、原材料供应紧张）。
- `prepayments`（预付款项）规模偏大或上升：说明对上游供应商的议价能力弱，需先付款才能采购。

**有息债务压力**
- `debt_ratio` 趋势：持续上升说明债务负担加重，偿债压力上升。
- 短期债务（`short_debt`）占比：短债比重高说明近期偿债压力大，需关注再融资风险。
- `fin_exp_ratio` 趋势：财务费用持续占收入较高比例，压缩核心利润空间，联系 R1 触发条件。

### 健康信号 vs 异常信号

| 信号类型 | 特征 |
|---|---|
| 健康 | `ap_vs_inventory` > 1 且稳定；`debt_ratio` 较低且稳定；`fin_exp_ratio` < 0.02；`prepayments` 占比小 |
| 观察 | `debt_ratio` 上升但 `fin_exp_ratio` 尚低；`ap_vs_inventory` 下降但未破 1 |
| 异常 | 四高异常（R1）：高有息负债 + 高财务费用并存；`prepayments` 大幅上升且应付账款下降，话语权显著弱化 |

### 框架解读

本模块对应张新民框架「底子—负债与权益端：经营性负债 vs 金融性负债」维度。经营性应付款（无息）是"造血型"融资，有息借款是"输血型"融资，两者结构决定资本效率与财务风险。

## 模块四：盈利质量

### 使用指标

| 指标键 | 来源 |
|---|---|
| `core_profit` | `metrics.md`：复杂公式（见 `metrics.md` 第一步） |
| `core_profit_margin` | `metrics.md`：`core_profit ÷ revenue` |
| `gross_margin` | `metrics.md`：`(revenue − cogs) ÷ revenue` |
| `selling_exp_ratio` | `metrics.md`：`selling_exp ÷ revenue` |
| `admin_exp_ratio` | `metrics.md`：`admin_exp ÷ revenue` |
| `rd_exp_ratio` | `metrics.md`：`rd_exp ÷ revenue` |
| `non_recurring_ratio` | `metrics.md`：`non_recurring_pnl ÷ net_profit` |

### 分析要点

**核心利润**
- `core_profit` 绝对值趋势：核心利润连续增长说明主业造血能力持续提升；连续下降需警惕。
- `core_profit_margin` 趋势：核心利润率下行是主业盈利能力受损的核心信号，结合 `non_recurring_ratio` 判断是否用非经常性收益掩盖（R3 触发逻辑）。

**毛利率**
- `gross_margin` 趋势：毛利率下行可能源于原材料成本上升、产品降价竞争或业务结构低毛利化。
- 毛利率与核心利润率同向下行：双重压缩，主业压力最大。
- 毛利率下行但核心利润率维持：说明三费管控有效，可抵消毛利压缩。

**三费结构**
- `selling_exp_ratio`：持续上升说明获客成本上升或营销依赖增强；过高且毛利率未提升，投入回报存疑。
- `admin_exp_ratio`：持续上升说明管理效率下降（规模不经济）。
- `rd_exp_ratio`：研发投入是否与收入匹配，过低可能影响长期竞争力，过高而核心利润持续亏损需关注资本化比例。
- `non_recurring_ratio`：≥ 0.30 结合利润率下行触发 R3（纸面利润依赖）；说明净利润质量存疑。

### 健康信号 vs 异常信号

| 信号类型 | 特征 |
|---|---|
| 健康 | `core_profit_margin` 稳定或上升；`gross_margin` 稳定；三费率控制合理；`non_recurring_ratio` < 0.10 |
| 观察 | `gross_margin` 小幅下行但 `core_profit_margin` 稳定；`non_recurring_ratio` 在 0.10–0.30 |
| 异常 | R3 触发（利润率下行 + 高非经常比）；毛利率与核心利润率双降；`selling_exp_ratio` 大幅上升侵蚀毛利 |

### 框架解读

本模块对应张新民框架「面子—利润表：核心利润去水分 + 结构化拆解」维度。核心利润是剔除金融损益、非经常项后的主业真实盈利，是评估「面子」成色最重要的指标。

## 模块五：现金质量

### 使用指标

| 指标键 | 来源 |
|---|---|
| `cash_conversion` | `metrics.md`：`cfo ÷ core_profit` |
| `cfo_to_net_profit` | `metrics.md`：`cfo ÷ net_profit` |
| `cfo` | 输入字段 |
| `cfi` | 输入字段 |
| `capex` | 输入字段 |
| `capex_intensity` | `metrics.md`：`capex ÷ revenue` |
| `cff` | 输入字段 |
| `dividend_payout` | `metrics.md`：`dividend_paid ÷ net_profit` |
| `dividend_paid` | 输入字段 |
| `new_equity_raised` | 输入字段 |

### 分析要点

**核心利润获现率**
- `cash_conversion` 理想状态 ≥ 1.0（核心利润全部转化为现金）；< 0.8 连续两年触发 R2（失血式增长）。
- `cfo_to_net_profit` < 1.0 说明净利润中有较多非现金成分（应收、递延等）。
- 两个获现率指标双低：现金质量极差，利润极可能含水分。

**投资回报验证**
- `cfi`（投资活动现金流）持续大额流出不一定是坏事，关键是后续是否带来营收/毛利率/核心利润改善。
- 复盘方法：查看大额 `capex` 年度后 1–2 年的 `revenue`、`gross_margin`、`core_profit` 变化；若无改善，历史投资效率存疑，未来减值风险上升。
- `capex_intensity` 趋势：持续上升但营收增速不匹配，资本投入效率下降信号。

**筹资与分红匹配度**
- `dividend_payout` ≥ 0.50 触发 R6（分红与再融资错配）；需同时检查 `new_equity_raised` 是否同期大额出现。
- 「一边高分红（`dividend_paid` 大）一边大额吸收投资（`new_equity_raised` 大）」：左手分右手借，本质是将股东资金套现或隐藏资金压力。
- `cff` 持续大额正值且 `debt_ratio` 上升：说明依赖外部融资维持经营，内生造血能力弱。

### 健康信号 vs 异常信号

| 信号类型 | 特征 |
|---|---|
| 健康 | `cash_conversion` ≥ 1 且稳定；`cfo_to_net_profit` ≥ 1；`capex` 后续带来业绩改善；分红率合理，无大额再融资错配 |
| 观察 | `cash_conversion` 在 0.8–1.0 间波动；`capex_intensity` 上升但营收也在增长 |
| 异常 | R2 触发（获现率连续 < 0.8）；R6 触发（高分红 + 高再融资）；`capex` 大额投入后业绩未改善；`cfo` 持续低于净利润 |

### 框架解读

本模块对应张新民框架「日子—现金流量表：核心利润获现率 + 投资活动现金流 + 筹资活动现金流与分红」维度。「日子」好不好，核心看经营活动能否持续产生真实现金，而非依赖外部输血维系。

## 模块六：风险结论

### 使用规则

R1–R7，来自 `rulebook.md`，包含触发条件、风险等级、建议复核项和风险含义。

### 分析要点

- 深度模式对每条触发规则展开「现象 → 机理 → 影响 → 建议复核项」四段式叙事。
- **现象**：直接引用第三步的触发证据（量化数值 + 年度）。
- **机理**：解释为何该组数字构成财务风险（引用 `rulebook.md` 的「风险含义」），不得自行发明新机理。
- **影响**：若风险成真，对资产负债表、利润表、现金流量表及投资者的实质影响（减值损失、流动性压力、盈利大幅下滑等具体方向）。
- **建议复核项**：完整引用 `rulebook.md` 中该规则的「建议复核项」，逐条列出，不得增减。

### 风险分类逻辑

| 风险类别 | 对应规则 | 核心财务影响 |
|---|---|---|
| 减值风险 | R5（商誉减值压力）、R7（减值计提不足） | 资产质量虚高，未来减值侵蚀净利润与净资产 |
| 流动性风险 | R1（四高异常）、R2（失血式增长） | 现金受限或持续失血，短期偿债压力 |
| 治理与舞弊预警 | R3（纸面利润依赖）、R4（渠道压货嫌疑）、R6（分红与再融资错配） | 利润质量存疑、收入造假风险、大股东利益输送 |

### 框架解读

本模块横跨底子-面子-日子三个维度，将量化触发结果转化为可操作的风险预警与复核清单。R1/R2/R7 偏重「底子」与「日子」风险，R3/R4 偏重「面子」风险，R5/R6 偏重「底子」与治理风险。

## 模块七：最终判断

### 使用结果

- 第三步：R1–R7 触发规则清单（highs / mids 计数）
- 第四步：三表勾稽结论（CC1/CC2/CC3 触发情况）
- 第五步：最终裁决（稳健 / 观察 / 谨慎 / 回避）

### 分析要点

**裁决结果展示**
- 直接引用第五步裁决结论，说明 highs/mids 数量与所触发规则名称。
- 裁决优先级：`highs ≥ 2 → 回避`，`highs == 1 → 谨慎`，`mids ≥ 2 → 观察`，否则 `稳健`（严格按 `cross_check_and_verdict.md` 逻辑，不得修改）。

**底子-面子-日子一致性论证**
- 将前五个模块的核心信号提炼为三条主线：
  - 底子（资产）：资产结构健康程度、资产质量、资本结构
  - 面子（利润）：核心利润率趋势、盈利质量
  - 日子（现金）：获现率、现金转化能力
- 判断三条主线是否一致：
  - **一致性强**：三表信号方向相同，财务质量可信。
  - **背离**：如面子靓丽（利润高增）但日子告急（获现率低），或底子扩张（资产大增）但面子/日子未跟上（收入/现金未增），是核心风险警示。结合 CC1/CC2/CC3 触发情况验证背离信号。

### 健康信号 vs 异常信号

| 信号类型 | 特征 |
|---|---|
| 底子-面子-日子一致（健康） | 资产优质、利润真实、现金充裕；三表勾稽未触发异常 |
| 面子虚胖（异常） | 净利润增长但核心利润率下行，非经常性依赖高；日子恶化（获现率 < 0.8）；CC3 触发 |
| 底子空洞（异常） | 资产规模扩大但收入未增；商誉比重高；在建工程大量沉淀；CC1 触发 |
| 日子失血（异常） | 获现率持续偏低；有息负债持续增长；经营现金流未能覆盖资本支出；R2 触发 |

### 框架解读

本模块是张新民框架的终极验证环节：底子（资产）、面子（利润）、日子（现金）三者的信号是否互相印证，是判断财务报表整体可信度的最高维度检验。三表背离往往是财务风险最直接的综合信号。
